# Final Project Summary

## Project Title

Fuzzy Inventory Model for Deteriorating Items

## Purpose

This project converts a published fuzzy inventory model into a reproducible computational repository using R.

The project demonstrates:

- documentation of the inventory model,
- sensitivity-analysis data entry,
- generation of CSV tables and plots,
- crisp model cost-function implementation,
- fuzzy model cost-function implementation,
- reported-vs-computed comparison for both crisp and fuzzy results.

## Source Model

The project is based on the paper:

Shaikh, T. S., & Gite, S. P. (2024).  
Fuzzy Inventory Model under Selling Price Dependent Demand and Variable Deterioration with Fully Backlogged Shortages.  
American Journal of Operations Research, 14, 87–103.  
DOI: https://doi.org/10.4236/ajor.2024.142005

## Model Features

The inventory model includes:

- selling-price-dependent demand,
- non-instantaneous deterioration,
- Weibull deterioration rate,
- fully backlogged shortages,
- crisp and fuzzy model formulations,
- hexagonal fuzzy numbers,
- Graded Mean Integration Representation Method for defuzzification.

## R Scripts

The repository contains three R scripts:

- `code/sensitivity_analysis_data.R`
- `code/crisp_model_cost_function.R`
- `code/fuzzy_model_cost_function.R`

## Generated Outputs

Generated CSV outputs are stored in:

`outputs/tables/`

Generated plots are stored in:

`outputs/plots/`

## Crisp Model Reproduction

The crisp model was implemented in R and compared with the reported paper values.

The computed results closely reproduce the reported values for:

- `t1`
- `T`
- total cost
- `q1`
- `q2`
- `Q`

The comparison file is:

`outputs/tables/crisp_model_reported_vs_computed.csv`

## Fuzzy Model Reproduction

The fuzzy model cost function was implemented in R using the GMIR-based computational structure.

The computed results closely reproduce the reported values for:

- `t1`
- `T`
- fuzzy total cost

The comparison file is:

`outputs/tables/fuzzy_model_reported_vs_computed.csv`

## Computational Notes

Two documentation issues were recorded:

1. The crisp numerical example reports `tm = 1.25`, but the MATLAB code uses `tm = 0.45`.
2. The paper reports the fuzzy demand coefficient with seven values, while the MATLAB fuzzy computation uses six values.

These issues are documented in:

`report/computational_notes.md`

## Reproducibility

The outputs can be regenerated by running:

```r
source("code/sensitivity_analysis_data.R")
source("code/crisp_model_cost_function.R")
source("code/fuzzy_model_cost_function.R")
```
